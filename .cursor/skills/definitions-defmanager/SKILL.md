# Definitions DefManager

## Назначение

Skill для безопасного добавления нового типа `Definition` в пайплайн загрузки:
- создание C#-класса;
- создание JSON-контента;
- подключение загрузки в `DefinitionsManager`.

## Критичное правило запуска

Выполняй этот Skill **только** при явном указании пользователя (оператора/программиста).

## Обязательное уточнение перед работой

Если в явном запуске Skill не указан формат хранения в менеджере, **остановись** и сначала задай уточняющий вопрос в чат:
- `single` (одиночный def через `LoadSingle<T>()`);
- `dictionary` (коллекция через `LoadCollection<T>()` и `Dictionary<string, T>`).

Пока не получен ответ, изменения в файлы не вносить.

## Рабочий процесс

1. Определи целевой модуль и менеджер (`Match3` или `Adventures`).
2. Найди существующий шаблон ближайшего def-типа по структуре полей.
3. Создай C#-класс дефа в `Assets/Modules/Definitions/Scripts/Implementation/...`.
4. Создай JSON в `Assets/Modules/Definitions/Resources/Definitions/...`.
5. Обнови `DefinitionsManager`:
   - добавь поле хранения (`T` или `Dictionary<string, T>`);
   - добавь метод загрузки;
   - добавь вызов метода в `LoadAll()` в правильном порядке.
6. Проверь, что:
   - `Id` формируется именем JSON-файла;
   - JSON соответствует полям C#-класса;
   - ссылки на другие def не ломаются при чтении.

## Ограничения проекта

- Не создавать/редактировать/удалять `MpGenerated` и MessagePack-генерацию.
- Не изменять файлы в `Assets/Scripts/Meta/SharedLogic/MergeGame/Models/Implementation/`.
- Не создавать и не редактировать Unity `.meta` файлы.

## Результат выполнения

После выполнения Skill агент обязан сообщить:
- какой формат был использован (`single` или `dictionary`);
- какие файлы добавлены/изменены;
- где именно подключена загрузка в `DefinitionsManager`.
